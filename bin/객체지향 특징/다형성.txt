- 하나의 객체 혹은 메서드가 여러가디 타입을 가질 수 있는 것
- 반대로, 단형성은 하나의 객체에 하나의 타입만 대응할 수 있음


오버 라이딩
- 부모 클래스로부터 상속받은 메서드를 자식 클래스에서 재정의해서 사용한다.
- 즉, 부모 클래스에 정의된 함수일지라도 자식 클래스에서 동일한 return, 동일한 이름, 동일한 접근제어자로 덮어쓰기(Override)를 하여 그 기능을 사용한다는 뜻

오버 라이딩의 제약사항
- 자식 클래스에서 재정의하는 메서드의 접근 제어자는 부모 클래스의 메서드보다 좁게 설정할 수 없다
-> protected -> public O / protected -> private X
- 자식 클래스에서 예외는 부모 클래스보다 더 큰 범위의 예외를 선언할 수 있다.
-메소드 오버라이딩은 상위 클래스의 메소드를 재정의 하는 것
- 쉽게 말해 부모 것을 덮어쓰는 것

오버 로딩
- 한 클래스 내에 이미 사용하는 이름의 매소드가 있더라도 특정 규칙을 지킨다면 동일한 이름의 메소드를 정의하도록 허용하는 기술술
- 하나의 클래스에 같은 이름의 함수를 여러 개 정의하고, 매개변수의 유형과 개수를 다르게 하여 사용한다.
-> 하나의 함수 이름으로 여러 매개 변수 타입을 넣어서 사용한다.
- 같은 이름으로 다양한 방식으로 쓰기

- 오버로딩 규칙
1. 메소드의 이름이 같아야 함
2. 매개 변수의 개수 또는 타입이 달라야 한다.
3. 매개 변수는 같고, 리턴 타입이 다를 때는 성립하지 않는다.
4. 오버로딩된 메소드들은 매개 변수로만 구분될 수 있다.


 @Override 
 - 어노테이션이라는 것으로 일반적인 주석과 다르게, 검증하는 기능을 함
 - 오버 라이딩을 검증하는 기능을 함
 - 코드상으로 검사했을 때 오버 라이딩이 실제로 시행되지 않았다면 컴파일 시 오류를 출력함
